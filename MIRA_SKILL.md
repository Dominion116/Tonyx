# Tonyx custom skill for @mira

Mira has no programmatic API — the only two-way bridge available to a third-party
project is a **deep link that pre-fills a chat message**, paired with a **custom
skill** that recognizes that message's shape and responds in character. This
document is the configuration to paste into Mira's custom-skill editor, plus the
reasoning behind it, so the "Ask Mira for a second opinion" button in Tonyx
produces a genuinely useful reply rather than a generic one.

## Install the skill

The skill is built and live. Anyone can add it to their own Mira with this one-tap link:

**https://t.me/mira?start=skill_vBuSTLd7**

Tapping it opens a chat with [@mira](https://t.me/mira) and offers to install "Tonyx Rebalance Reviewer"
directly — no manual configuration needed. This is the link to share in the bot's description,
the `/start` message, and the hackathon submission.

## How the bridge works end to end

0. The user installs the skill once via the link above (prompted from Tonyx's own
   `/start` message or bot description).
1. A user reviews a Tonyx rebalance proposal (Telegram bot or web dashboard).
2. They tap **"🔮 Ask Mira for a second opinion"**.
3. This opens `https://t.me/mira?text=<encoded message>` — built by
   [`buildAskMiraDeepLink`](packages/shared/src/integrations/mira-link.ts), the
   single shared formatter used by both surfaces.
4. The chat compose box is pre-filled with a `[TONYX PROPOSAL]`-tagged, plain-language
   summary of the route, its economics, and Tonyx's own confidence and reasoning.
5. The user sends it. The skill below recognizes the `[TONYX PROPOSAL]` tag and
   has Mira respond as a second-opinion reviewer rather than a generic assistant.

## Skill configuration

**Name:** `Tonyx Rebalance Reviewer`

**Slug / command:** `/tonyx_review` (also auto-triggers on the phrase `[TONYX PROPOSAL]`,
so the deep link works whether or not the user types the slash command)

**Trigger phrases:**
- `[TONYX PROPOSAL]`
- `tonyx proposal`
- `/tonyx_review`

**Behavioral instructions** (paste into the skill's instruction/prompt field):

```
You are reviewing a yield-rebalancing proposal generated by Tonyx, an autonomous
TON yield-optimization agent. Messages tagged "[TONYX PROPOSAL]" follow a fixed
plain-language format with these fields, in order:

  Route: <origin pool> -> <destination pool>
  Amount: $<amount routed, USDT>
  APR: <destination pool APR, %>             (omitted if unavailable)
  Est. yield: $<amount>/day
  Tonyx confidence: <percent>                 (Tonyx's own deterministic confidence score)
  Tonyx says: "<Tonyx's own plain-language explanation>"

When you see this format, do NOT treat it as a generic question. Instead, act as
an independent second-opinion reviewer:

1. Sanity-check the math: does the estimated yield plausibly follow from the
   amount and APR? Flag it if something looks off (e.g. an APR that looks too
   good to be true for a liquidity pool).
2. Weigh the route itself: is moving funds from the origin to the destination pool
   reasonable given the stated APR and amount? Would you size it differently?
3. React to Tonyx's own confidence and explanation — agree, disagree, or add a
   nuance it may have missed (e.g. impermanent loss risk, pool concentration,
   whether the gain is large enough to be worth the slippage).
4. Give a clear, short verdict: would you execute this route as-is, adjust it, or
   hold off — and one sentence on why.

Keep the whole reply conversational and under ~120 words. You are a second set of
eyes, not a replacement for Tonyx's own engine — be direct about where you agree
and where you'd push back.
```

**Example trigger message** (what the deep link pre-fills):

```
[TONYX PROPOSAL]
Route: idle USDT -> STON.fi USDT/TON
Amount: $250.00
APR: 18.40%
Est. yield: $0.1260/day
Tonyx confidence: 62%
Tonyx says: "Estimated yield clears the policy floor with a modest margin — APR is solid for the size, but the edge is thin."

Mira, what is your take on this route — worth executing, or is there a smarter move here?
```

**Example desired Mira reply:**

```
Mira, here's my read: the math checks out — at 18.4% APR on $250 you're earning
about 12.6¢/day, and after Tonyx's 10¢ fee you keep ~2.6¢. That's real, but thin —
you're handing over ~80% of the daily yield to the execution fee. Tonyx's 62%
confidence feels right: it clears the floor, but barely. I'd either wait for a
larger idle balance (so the fixed fee is a smaller bite) or a meaningfully higher
APR before pulling the trigger. Not a bad route, just not an urgent one.
```

## Why this design

- **`[TONYX PROPOSAL]` as the trigger, not a slash command alone** — Telegram's
  `?start=` deep-link parameter is capped at ~64 characters and `[A-Za-z0-9_-]`
  only, far too small for a real proposal. `?text=` has no such limit and lets us
  hand Mira the full picture in natural language, so the link is useful with or
  without the skill installed.
- **One shared formatter** — `buildAskMiraDeepLink` lives in `@tonyx/shared` and
  is used by both the Telegram bot (`apps/api/src/telegram/commands/rebalance.ts`)
  and the web dashboard (`apps/web/components/quote/quote-modal.tsx`), so the
  message format the skill pattern-matches on can never drift between surfaces.
- **Asking for a verdict, not a chat** — the instructions push Mira toward a
  structured, opinionated review (math check, route judgment, agree/disagree,
  verdict) rather than open-ended conversation, which is what makes the "second
  opinion" framing land as genuinely useful rather than decorative.
